# Terra Form Textarea Upgrade Guide

## Changes from version 4 to version 5

### Props

#### Removed
* Removed `placeholder` prop from `Textarea` and  `TextareaField`.

### Changes to CSS Custom Properties

#### Removed
* --terra-form-textarea-placeholder-color
* --terra-form-textarea-placeholder-font-style
* --terra-form-textarea-placeholder-disabled-color
* --terra-form-textarea-placeholder-disabled-font-style

##### Steps to Upgrade
To upgrade from 4.0 to 5.0 each existing Textarea and TextareaField must be reviewed and updated to ensure no `placeholder` prop is passed to it. This is done to follow WCAG ([for more info](https://www.w3.org/WAI/GL/low-vision-a11y-tf/wiki/Placeholder_Research)).

* `Textarea` component will not be supporting `placeholder` prop and `native placeholder`. If consumers want to provide a detailed text guidance they can upgrade to `TextareaField`.

* `TextareaField` component will not be supporting `placeholder` prop and `native placeholder`. But consumers may utilize the `help` text field properties to provide similar guidance to the user using `help` prop.

## Changes from version 3 to version 4

### Props

#### Updated
* Form Fields allowed the `label` to be a node, which potentially breaks the display when injecting other non-inline-only text elements (terra-text) or has the potential to completely break accessibility if something non-text based would be passed in.
* `label` prop's type changed from node to string.

##### Steps to Upgrade
To upgrade from 3.0 to 4.0 each existing Textarea must be reviewed and updated to ensure only a string is being passed to the `label` props. Solutions passing node data types need to be reworked to provide only a string.
